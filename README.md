### 핵심내용

제안배경 및 목적

현대 청년들은 수많은 지원사업, 훈련, 채용 정보 속에서도 “나에게 맞는 기회”를 찾기 어려움. 그 이유는 개인의 노력 이전에 **지역·학력·전공·소득** 등 **사회적 구조에 따른 기회 노출 격차**가 존재하기 때문임.

Fair Opportunity.AI는 이러한 **기회 불평등을 데이터로 진단하고**, AI를 통해 **개인별로 공정한 성장 경로를 제시**하는 플랫폼임.

> “기회가 없는 것이 아니라, 닿지 않는 것이다.”
> 
> 
> → **기회의 불평등(FOI)** → **성과의 불평등(EOI)** → **가치의 불일치(LOI)**
> 
> 이 세 가지를 연결해 “공정한 성장(FGS)”을 구현.
> 

---

### 아이디어의 기원

이 아이디어는 **세 가지 흐름**에서 출발함.

1. **사회구조적 불평등 분석의 전통**

UNDP *Human Development Index*, OECD *Better Life Index*, 미국 *Opportunity Index* 등 다차원 지표에서 영감을 얻음.

기존 지표가 ‘기회의 존재’까지만 다뤘다면, **FOI(Fair Opportunity Index)** 는 “존재 + 접근 가능성”을 함께 평가.

→ *개인의 선택 이전, ‘출발선의 공정성’을 수치화.*

1. **일자리의 질과 지속가능성에 대한 새로운 관점**

OECD/EU의 *Employment Quality Index*는 임금·안정·일·삶의 균형을 평가함.

**EOI(Employment Outcome Index)** 는 이를 확장하여 임금 수준, 근속, 만족, 성장 가능성까지 통합 측정.

→ *‘기회의 문을 연 뒤의 현실’을 계량화.*

1. **개인가치 기반의 성장 모델**

Self-Determination Theory, PERMA 모델에서 착안.

**LOI(Life Orientation Index)** 는 개인의 **워라밸, 성장, 안정, 자율성**을 수치화하고, 이를 바탕으로 EOI의 가중치를 동적으로 조정.

→ *“모두에게 좋은 일자리”가 아니라 “나에게 공정한 결과”.*

또한 세계은행 *Human Capital Index*처럼, **기회(FOI) × 성과(EOI) × 가치(LOI) = 공정한 성장(FGS)** 구조를 설계.

즉, 사회가 주는 기회와 개인의 성향이 결합할 때 비로소 ‘의미 있는 성장’이 가능.

> 기존의 AI가 “당신은 고소득 직무에 적합하다”고 말한다면,
> 
> 
> **Fair Opportunity.AI는 “당신의 가치에 따라, 가장 공정하고 지속가능한 경로는 이것입니다.”** 라고 말하는 것.
> 

### 세부내용

### (1) 활용 데이터 및 AI 모델

| 구분 | 역할 | 설명 |
| --- | --- | --- |
| FOI | 기회 접근성 계산 | 지역·소득·학력·전공 기반 (공공데이터) |
| EOI | 직무별 결과 품질 | 임금·근속·만족도 등 (워크넷, 고용정보원) |
| LOI | 개인 가치 설문 | 워라밸·성장·안정·자율성 |
| FGS | 종합 성장 점수 | AI 예측 모델로 산출 |
- **데이터 분석 AI**: FOI·EOI 계산 (정규화 + 결측 보정)
- **추천 AI**: 유사도 + 격차보정 가중치 + 정책자격 필터
- **설명형 AI**: 추천 사유를 자연어로 생성
- **예측 AI**: 목표 달성 확률(FGS) 시뮬레이션

---

### (2) 지표 정의 및 산식

| 지표 | 정의 | 주요 구성 요소 | 산식 |
| --- | --- | --- | --- |
| **FOI**(Fair Opportunity Index) | 사회적 환경에서의 기회 접근성 | E(고용), T(훈련), I(정보) | `0.4E + 0.35T + 0.25I` |
| **EOI**(Employment Outcome Index) | 일자리의 결과 품질 | W(임금), S(안정), G(성장·만족) | `0.5W + 0.3S + 0.2G` |
| **LOI**(Life Orientation Index) | 개인 가치·선호 | 워라밸, 성장, 안정, 자율성 | `0.3WLB + 0.3Growth + 0.2Security + 0.2Autonomy` |
| **FGS**(Fair Growth Score) | 공정 성장 가능성 | FOI, EOI_personal, LOI | `0.25FOI + 0.35EOI_personal + 0.25(FOI×EOI_personal/100) + 0.15LOI` |

### (3) 예시 시뮬레이션

| 입력 항목 | 값 |
| --- | --- |
| 지역 | 전북 정읍 |
| 학력 | 전문대 |
| 전공 | 디자인 → 데이터분석 전환 |
| 성향 | 워라밸형 |
| 계산결과 | FOI=54 / EOI=58 / LOI=82 / EOI_personal=62 |
| **FGS = 약 56점** |  |
| 개선조치 | 무료 온라인 강좌 +8 FOI, NCS 자격 +7 EOI_personal |
| **예상 FGS = 68~70점** |  |

> “온라인 과정 부족 → 무료 비전공 과정 추천”
> 
> 
> “NCS 데이터 기초 자격 → 성과 점수 +7 상승”
>

### (4) 기술 구조 (MVP DB 설계)

**데이터 구조 요약**

| 계층 | 테이블 예시 | 내용 |
| --- | --- | --- |
| 기준(dim) | `dim_region`, `dim_education`, `dim_occupation`, `dim_time` | 지역/학력/직무/시점 |
| 지표(fact) | `fact_foi_metrics`, `fact_eoi_metrics` | 공공데이터 기반 FOI/EOI |
| 사용자(user) | `user_profile`, `user_loi_assessment`, `user_scores` | 개인 입력/설문/산출 점수 |

**ERD 핵심 흐름**

> dim_* → fact_* → user_*
> 
> 
> (공공데이터 기반 사회지표 → 개인 입력 → AI 결과(FGS))
> 

**데이터 파이프라인**

- 수집: KOSIS·HRD-Net·워크넷·NIA API
- 정제: 단위 통일 → 0~100 정규화 → 결측치 평균 보정
- 버전관리: `source_version`, `score_version`
- 주기: FOI/EOI 분기, LOI 실시간, FGS 상시

### (6) 창의성 및 구현 가능성

- 기존 AI가 **‘고소득 중심 경로’**를 제시한다면, 본 서비스는
    
    **“성공의 정의를 표준화하지 않는다.”**
    
- 공공데이터 + 개인가치 데이터 융합을 통한 **사회구조·심리모델 결합형 AI**
- 워크넷/HRD-Net API 연동으로 **실행(신청)까지 연결 가능한 실제 구현 모델**
